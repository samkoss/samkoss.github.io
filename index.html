I am a Junior Computer Science and Mathematics major with a Cybersecurity minor at the University of Delaware. This semester I took Secure Software Design that aimed to teach developers how to design programs from start to finish with security in mind.

Secure Software Design

This upperlevel course consists of a variety of projects that require consideration of the SEI CERT C Coding Standard and CERT C++ Coding Standard to prevent C/C++ related vulnerabilities. Groups worked together as blue teams to develop secure programs while, red teams searched for exploits and unsafe practices in existing code.

The Projects

Markovian Tweet Generator

Given a text file, this program generates a tweet in the writing style of the text. The deliverable was searched for Memory Management (MEM) and Characters and Strings (STR) rules and recommendations that were followed or broken from the SEI CERT C Coding Standard and CERT C++ Coding Standard.

Polynomial Addition

With two polynomials stored in files, the program adds the two into a resulting polynomial. Memory Management (MEM) and Characters and Strings (STR) rules and recommendations were considered for the entirety of development and in code reviews by peers.

Command-Line Minesweeper

Given a board size and desired number of bombs, generates a minesweeper board and playable game based on moves players can make from the command line. This program was developed to follow the rules or recommendations from Arrays (ARR) Expressions (EXP). It was additionally reviewed afterward for the same rules.

Gee-Mail Secure Messaging

This program is a secure messaging application in the command line using an SQLite3 database in the back-end. Features include user registration and authentication, and the ability to write and read messages to and from other users in the database. The featured rules and recommendations are Input Output (FIO) and Error Handling (ERR). In code reviews, all rules from the SEI CERT C/C++ Coding Standard were used to evaluate the code security.
